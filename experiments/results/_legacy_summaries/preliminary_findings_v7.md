# Предварительные выводы v7 — после правки K как параметра среды

**Дата:** 2026-05-01.
**Главное изменение по сравнению с v6:** $K$ перестал быть глобальным гиперпараметром $K=3$. Теперь $K_t = \min(K_{\max}, |J_t| - 1)$ при $|J_t| \geq 2$, $K_t = 1$ при $|J_t| = 1$, $K_{\max} = 3$. Для Mobius (3 параллельных зала) это даёт $K_t = 2$ во всех слотах: политика всегда отсекает один зал. До правки compliance-механизм на Mobius вырождался в ранжирование без отсечения, что снимало эффект параметра $\alpha_{\text{compliance}}$. Все основные прогоны переснятые на 01.05.

## Stylized facts replication

| Факт | Метрика | Mobius (1200 агентов) | Воспроизводится? |
|---|---|---|---|
| 1. Pareto-attendance | top-20 talks share | Cosine: 52.7%, MMR: 55.4%, Capacity-aware: 49.5%, LLM-ranker: 49.3% | ✓ для Cosine/MMR (KS p<0.01); ✗ для Capacity-aware (намеренно размывает) |
| 2. Time-of-day decline | slope of attendance vs slot_idx | Все политики: slope ≈ −0.05, p < 0.005 | ✓ Воспроизводится для всех |
| 3. Track-affinity | normalized H of categories per user | normalized_H ≈ 0.65, random ≈ 1.47 | ✗ Не воспроизводится (агенты ходят более разнообразно) |

**Negative result для track-affinity** методологически честный: указывает, что в модели агента отсутствует инерция тематического интереса. Известное ограничение симуляторов с retrieval-only memory без topical preference reinforcement; зафиксировано в направлениях дальнейшей работы.

**Итог:** 2 из 3 стилизованных фактов воспроизведены при p < 0.01.

## Главный научный вывод (для Главы 4)

> «В обстановке малого каталога ресурсов с жёсткими ограничениями вместимости параллельных сессий простая эвристическая политика с явным учётом текущей заполненности залов (Capacity-aware) полностью устраняет переполнения в выборах участников ($\text{OF}_{\text{choice}} = 0{,}000$) при потере менее 4% средней пользовательской полезности по сравнению с косинусной политикой ($0{,}331$ vs $0{,}351$). Сочетание учёта вместимости с механизмом тематического разнообразия (Capacity-aware MMR) даёт минимальную дисперсию заполненности залов ($0{,}008$, на два порядка ниже косинусной политики). Результат устойчив через два конференционных набора разной структуры (Mobius, Demo Day) и переносится на адаптированный набор MovieLens 1M (Spearman $\rho = +0{,}85$, $p = 0{,}004$ для mean_overload_excess; $\rho = +0{,}90$, $p < 0{,}001$ для hall_utilization_variance). Граница применимости определяется параметром восприимчивости пользователя к рекомендации: при $\alpha_{\text{compliance}} \geq 0{,}7$ преимущество политик с учётом вместимости статистически значимо и существенно (соотношение overflow до 59× при $\alpha = 0{,}9$); при $\alpha \leq 0{,}5$ преимущество не превышает 1,4× в силу существенной доли участников, игнорирующих выдачу политики и принимающих решения исходя из всего множества кандидатов. Симулятор воспроизводит два из трёх стилизованных фактов поведения участников конференций (Парето-распределение посещаемости и снижение посещаемости к концу программы) при $p < 0{,}01$; третий факт (тематическая инерция выбора) не воспроизводится, что интерпретируется как ограничение модели агента и относится к направлениям дальнейших исследований.»

## Что осталось до полной согласованности

- Inter-slot chat full на 1200 агентов — не сделано, в направлениях дальнейшей работы.
- Multi-agent PPO с batch_size=1200 — не сделано, в направлениях дальнейшей работы.
- Sensitivity по w_fame ∈ {0.0, 0.15, 0.3, 0.5} — желательно для закрытия атаки «подкрутка fame».
- Протокол статистики (Wilcoxon + Bonferroni) — желательно для главы 4.
