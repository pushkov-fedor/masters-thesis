# Claude Code: проектные инструкции

## ⚠ Точка входа в проект

Когда пользователь просит «ознакомься с проектом» (или эквивалент — «погрузись в проект», «вспомни проект», «get up to speed», «введи в курс дела») — **первым делом прочитать `PROJECT_OVERVIEW.md` в корне репо**. Это сжатый снимок всего проекта (модель, код, результаты, текст ВКР, открытые вопросы предзащиты), специально подготовленный как точка входа, чтобы не перечитывать всё содержимое репо с нуля.

После `PROJECT_OVERVIEW.md` при необходимости углубляться в:
- `PROJECT_STATUS.md` — текущий статус, дедлайны, стоп-лист;
- `PROJECT_DESIGN.md` — формальная постановка;
- `PIVOT_IMPLEMENTATION_PLAN.md` — план реализации (r5, спринт закрыт);
- `experiments/results/report_mobius_2025_autumn_full.md` — финальный экспериментальный отчёт;
- `thesis/ПушковФВ_ВКР.pdf` — текст ВКР после антиплагиата.

## Иерархия документов

1. `PROJECT_OVERVIEW.md` — навигационный снимок, точка входа.
2. `PROJECT_STATUS.md` + `PROJECT_DESIGN.md` — авторитетные документы по теме, тезису, фрейму, валидации.
3. `PIVOT_IMPLEMENTATION_PLAN.md` — план; ревизии r1→r5.
4. Код в `experiments/` — источник фактов о реализации, не постановки.
5. `docs/spikes/` — методические memo принятых решений.
6. `materials/_legacy/`, `thesis/chapter3.tex` (старый), `experiments/scripts/_legacy_scripts/` — НЕ использовать как актуальный контекст.

Если `PROJECT_OVERVIEW.md` противоречит документам 2–5 — верить им, не снимку.

## Жёсткие правила

- Не возвращать в защитную логику: B1 / accuracy@1, cross-domain Spearman, MMR-как-победитель, UMass-как-конференция, CoSPLib-как-attendance, jug-rec-sys как защищаемый результат. Полный стоп-лист — `PROJECT_STATUS.md` §5.
- Текст ВКР после антиплагиата (08.05.2026) не переписывается до защиты; крупные правки откладываются на финальную версию.
- Python через `uv venv` / `uv pip`; не ставить пакеты глобально без явного разрешения.
- Академический стиль текста: безличные конструкции, минимум англицизмов.
