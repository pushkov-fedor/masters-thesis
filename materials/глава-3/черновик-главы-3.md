# Глава 3. Реализация программного решения (черновик)

> Статус: рабочий черновик, итеративная сборка. Не финальный текст. Соответствует версии файла thesis/chapter3.tex.

## 3.1 Архитектура программного решения и контекст интеграции

Разработанная в рамках работы система реализована как самостоятельный программный комплекс, ориентированный на проведение сравнительного исследования рекомендательных политик в условиях, описанных в главе 2. Архитектура системы организована вокруг трёх связанных подсистем: подсистемы синтетических данных и моделей релевантности, симуляционной среды с двумя слоями валидации и набора сравниваемых рекомендательных политик. Управляющие сценарии оформлены как набор автономно запускаемых программ, обеспечивающих воспроизводимость отдельных этапов вычислительного эксперимента.

Программный комплекс не является самостоятельным сервисом, готовым к промышленной эксплуатации. Его прикладной контекст определён существующей инфраструктурой профильного партнёра — открытым программным проектом коллектива научного руководителя, расположенным в сопровождающем репозитории. Существующая инфраструктура включает векторную базу данных Qdrant, REST-сервис рекомендаций на основе содержательной близости эмбеддингов, сервис расписания и пользовательский клиент в виде телеграм-бота. Сервис рекомендаций возвращает упорядоченные по убыванию косинусной близости списки докладов; согласование между пользователями и учёт ограничений вместимости в существующей системе не реализованы. Реализованный в работе слой управляющих политик предлагается как надстройка над существующим сервисом косинусной близости: политика принимает на вход список кандидатов с оценками близости и текущим состоянием загрузки залов и возвращает переупорядоченный список с возможностью отбрасывания позиций, превышающих порог загрузки. Прикладная интеграция выходит за рамки настоящей работы и относится к направлениям дальнейших исследований.

Общая последовательность вычислений, реализованных в системе, представлена ниже. Программа конференции и пользовательская популяция загружаются из подготовленных файлов; для каждой пары «профиль участника, описание доклада» вычисляется значение релевантности; рекомендательная политика на каждом тайм-слоте формирует список рекомендаций фиксированного размера для каждого участника с учётом текущей загрузки залов; имитационная модель отклика возвращает фактический выбор каждого участника; на основе агрегированных по эпизоду решений вычисляются целевые метрики. Цикл повторяется для всех тайм-слотов программы; результаты прогона сохраняются и используются на этапе сравнительного анализа политик в главе 4.

Отдельной частью комплекса является среда автономных агентов, в которой отклик пользователя реализуется не параметрической функцией, а порождающим вызовом большой языковой модели. Эта среда используется во втором слое валидации; её устройство описано в подразделе 3.2.2.

## 3.2 Реализация симуляционной среды

Симуляционная среда реализована как набор согласованных модулей на языке Python с использованием библиотек численных вычислений NumPy и SciPy. Структуры данных, описывающие программу конференции, оформлены в виде классов с фиксированной семантикой полей: класс доклада хранит идентификатор, заголовок, ссылку на зал, идентификатор тайм-слота, тематическую категорию, текст краткого описания и нормированный эмбеддинг описания; класс зала хранит идентификатор и фактическую вместимость; класс тайм-слота хранит идентификатор, временную метку и список идентификаторов докладов, проходящих в этом слоте; класс конференции содержит словари докладов и залов и упорядоченный список тайм-слотов. Программа конференции загружается из файла в формате JSON, эмбеддинги докладов хранятся в отдельном бинарном файле и привязываются к идентификаторам докладов на этапе загрузки.

### 3.2.1 Параметрический симулятор

Первый слой симуляционной среды реализует параметрическую модель отклика пользователя в соответствии с постановкой подраздела 2.1. На каждом тайм-слоте для каждого пользователя политика возвращает упорядоченный список рекомендаций длины K; имитационная модель отклика реализует мультиномиальное логистическое распределение по альтернативам списка с возможностью отказа от выбора, после чего обновляет загрузку залов до состояния, видимого следующим пользователям того же слота.

Конфигурация симулятора задаётся параметрами K (длина выдачи), τ (температура распределения отклика), λ (вес штрафа за переполнение), p_skip (базовая вероятность отказа от выбора). В дополнение к базовой постановке в реализованную модель отклика введены два параметра, отвечающие за усиление реалистичности экспериментальной обстановки: вес признака популярности доклада w_fame и параметр восприимчивости пользователя α_compliance. Описание этих параметров приведено в подразделе 3.5.

Внутри симулятора реализована возможность подмены функции релевантности: вместо штатной функции косинусной близости можно передать произвольную функцию, принимающую на вход эмбеддинги профиля и доклада и возвращающую скалярное значение. Эта возможность используется для подключения обучаемой модели предпочтений, описанной в подразделе 3.3.

### 3.2.2 Симуляционная среда на основе автономных агентов

Второй слой симуляционной среды реализует независимую модель отклика пользователя на основе больших языковых моделей. Каждый агент представлен экземпляром класса генеративного агента, объединяющим текстовый профиль пользователя, нормированный эмбеддинг профиля, систему памяти с извлечением по релевантности, модель личности на основе пятифакторной модели «Большая пятёрка», состояние накопленной усталости и асинхронный вызов к большой языковой модели через единый интерфейс. На каждом тайм-слоте каждый агент получает на вход список кандидатов слота с описаниями, текущую загрузку залов, рекомендацию политики, агрегированный социальный сигнал от агентов из соседства в графе знакомств и возвращает решение в виде идентификатора выбранного доклада или отказа от выбора с кратким объяснением причины.

Граф знакомств реализован как граф Уоттса—Строгатца с заданным числом ближайших соседей и параметром перепосева; используется для имитации малой когорты, в которой решения близких знакомых становятся видимыми пользователю. Социальный сигнал, передаваемый агенту, формируется как краткое перечисление залов, в которые направились знакомые в текущем тайм-слоте. Внутри одного тайм-слота агенты обрабатываются последовательными группами заданного размера, что обеспечивает кумулятивное обновление загрузки залов и социального сигнала по мере поступления решений.

Дополнительно во втором слое реализован механизм обмена впечатлениями между тайм-слотами. После завершения тайм-слота заданная доля посетивших агентов формирует короткое текстовое сообщение о посещённом докладе через вызов большой языковой модели; сообщения накапливаются в общем пуле с привязкой к эмбеддингу доклада и временной метке. На очередном тайм-слоте перед принятием решения каждый агент получает несколько сообщений из пула, наиболее близких по эмбеддингу к кандидатам текущего слота. Этот механизм соответствует требованию учёта косвенного социального влияния, формирующегося во время мероприятия в неформальных каналах коммуникации участников; его влияние на устойчивость выбранной политики оценивается в исследовании влияния отдельных компонентов метода в главе 4.

Все вызовы к большой языковой модели проходят через асинхронный интерфейс с ограничителем числа одновременных запросов и автоматическим повтором при превышении тайм-аута. Стоимость использования модели рассчитывается по фактическому числу токенов запроса и ответа и фиксируется в выходных данных эпизода.

## 3.3 Реализация моделей релевантности

Функция релевантности реализована в двух вариантах. Базовый вариант — косинусная близость нормированных эмбеддингов профиля участника и описания доклада. Эмбеддинги получены однократным вызовом модели представления текстов и сохранены в бинарных файлах с привязкой к идентификаторам.

Дополнительный вариант реализован как обучаемая модель предпочтений на основе градиентного бустинга. Обучающая выборка получена однократным запросом к большой языковой модели для всех пар «профиль, доклад» с целью получения скалярной оценки степени интереса участника к докладу; полученная матрица предпочтений используется как целевая переменная. Признаки модели сформированы как покомпонентное произведение нормированных эмбеддингов профиля и доклада. Модель обучена на восьми девятых полученной выборки, оставшаяся часть использована для контроля качества; на отложенной выборке достигнут коэффициент ранговой корреляции Пирсона около 0,79. Обученная модель сохранена в формате pickle и подключается к параметрическому симулятору через подмену функции релевантности; для ускорения вычислений предусмотрено предварительное вычисление и кэширование значений для всех пар «профиль, доклад» в начале эпизода.

Использование двух функций релевантности позволяет в главе 4 отделить наблюдения, обусловленные выбором конкретной модели близости, от наблюдений, отражающих структуру задачи независимо от модели близости.

## 3.4 Реализация рекомендательных политик

В главе 2 в составе предлагаемого метода предусмотрено сравнение шести политик различной природы. На этапе реализации спецификация политик расширена до двенадцати, в том числе с целью более детального изучения границ применимости простых и сложных подходов в обстановке малых каталогов. Дополнительные политики добавлены без изменения общего методологического дизайна и относятся к категориям, рассмотренным в концептуальном виде в главе 1.

Все политики реализованы в виде вызываемых объектов с единым интерфейсом: на вход политика принимает текущего пользователя, текущий тайм-слот, ссылку на конференцию и словарь состояния симуляции, включающий загрузку залов и функцию релевантности; на выход возвращается список идентификаторов рекомендованных докладов длины K или меньше, упорядоченный по убыванию приоритета.

Краткое описание реализованных политик приведено ниже.

**Случайная политика** формирует список из K случайно выбранных без возвращения докладов слота. Используется в качестве нижней опорной точки сравнения.

**Политика косинусной близости** возвращает K докладов слота с максимальной косинусной близостью эмбеддинга описания к эмбеддингу профиля пользователя. Соответствует поведению существующего сервиса рекомендаций партнёра.

**Политика максимально маргинальной релевантности** формирует список последовательным жадным выбором: на каждом шаге к списку добавляется доклад, максимизирующий выпуклую комбинацию косинусной близости к профилю и удаления от уже выбранных докладов с весом β. Алгоритм соответствует процедуре, известной в литературе как Maximal Marginal Relevance.

**Политика с учётом вместимости** ранжирует доклады слота по уменьшаемой косинусной близости с вычитанием взвешенного штрафа за текущую загрузку зала; залы с загрузкой выше заданного порога исключаются из выдачи. Реализует решение, выдвигаемое организатором мероприятия как простое содержательное правило «не направлять участника в почти полный зал».

**Политика с учётом вместимости и максимально маргинальной релевантности** объединяет схему максимально маргинальной релевантности с штрафом за текущую загрузку залов и порогом отсечения переполненных залов.

**Политика на основе детерминантных точечных процессов** формирует выдачу через жадную максимизацию определителя на матрице ядра, согласующего сходство с профилем и попарное сходство между докладами; обеспечивает разнообразие выдачи через свойства определителя.

**Калиброванная политика** формирует выдачу через минимизацию взвешенной суммы дивергенции Кульбака—Лейблера между распределением категорий в выдаче и распределением категорий, предпочитаемых пользователем, и отрицательной суммарной релевантности.

**Последовательная политика** использует историю предыдущих решений пользователя в этом эпизоде как контекст: к косинусной близости с эмбеддингом профиля добавляется взвешенная косинусная близость со средним эмбеддингом ранее посещённых докладов в скользящем окне. Подход воспроизводит идею, известную в литературе как контекстная последовательная рекомендация.

**Политика на основе графовой нейросети** формирует выдачу через двухслойное распространение сообщений по графу «пользователь—доклад» с порогом сходства; реализована как численная процедура без обучаемых параметров и используется для сопоставления влияния структуры графа сходств на качество выдачи.

**Политика, обучаемая методом проксимальной оптимизации** реализует подход, описанный в главе 2 как обучаемая политика марковского процесса принятия решений с ограничениями. Эпизод обучения соответствует одной полной программе конференции; на каждом тайм-слоте политика последовательно рекомендует группе пользователей одно из доступных пространств действий, состояние включает эмбеддинг профиля, нормированную загрузку залов слота, индикаторный вектор тайм-слота, оставшуюся долю необработанных пользователей и максимальную популярность докладов в зале; маска действий исключает залы, превысившие порог загрузки. Награда формируется как сумма релевантности и поправки за популярность за вычетом штрафа за переполнение. Обучение проведено на двухстах тысячах шагов сценария партнёра с использованием библиотеки Stable Baselines 3 и расширения для маскируемых действий.

**Политика на основе ранжирования большой языковой моделью** обращается к большой языковой модели через единый интерфейс с запросом упорядочить кандидатов слота по соответствию профилю; реализованы две версии — без учёта состояния и с передачей в запрос текущей загрузки залов; результаты вызовов кэшируются по парам «пользователь, слот» для воспроизводимости и контроля стоимости.

## 3.5 Управление параметрами реалистичности экспериментальной обстановки

Базовая постановка задачи в главе 2 предполагает однородную модель влечения пользователя к докладам, опирающуюся только на содержательную релевантность. На этапе реализации экспериментальной обстановки введены два дополнительных параметра, обеспечивающих более полное соответствие реалистичной картине поведения участников.

Первый параметр — признак популярности доклада, агрегирующий пять структурных характеристик описания доклада в программе конференции: длина текста описания, число со-докладчиков, принадлежность доклада к активно представленным в программе тематическим категориям, формат тайм-слота (одиночное выступление как формат основного доклада), упоминание крупной компании в составе со-докладчиков. Каждая характеристика нормируется и взвешивается; итоговое значение приведено к отрезку [0, 1]. Расчёт признака реализован однократным запуском обработки программы конференции и сохраняется в отдельном файле, привязываемом к программе на этапе загрузки. В функции отклика пользователя эффективное значение влечения определяется как выпуклая комбинация значения релевантности и значения признака популярности с весом w_fame. Введение признака позволяет воспроизвести ситуацию, в которой отдельные доклады собирают аудиторию преимущественно в силу структурных особенностей программы, независимо от индивидуальных предпочтений участника; эта ситуация существенна для оценки устойчивости политик к концентрации спроса в небольшом числе позиций.

Второй параметр — параметр восприимчивости пользователя, задающий вероятность, с которой пользователь принимает решение строго из выдачи политики. С дополнительной вероятностью пользователь принимает решение из всех докладов слота на основе эффективного значения влечения, без учёта рекомендации; пороговая вероятность задаётся параметром α_compliance в отрезке [0, 1]. Введение параметра позволяет оценить границу применимости рекомендательной политики при разных уровнях доверия пользователей к подсказке системы. Анализ поведения политик при изменении параметра восприимчивости проводится в главе 4 в составе исследования устойчивости результатов.

## 3.6 Тестовые наборы данных

Основная экспериментальная обстановка построена на основе двух конференций, программа которых получена в открытом доступе. Первая конференция — Mobius 2025 Autumn, профильное мероприятие в области мобильной разработки; программа включает сорок докладов в трёх параллельных залах с фактическими вместимостями шестьсот, триста пятьдесят и двести пятьдесят мест на шестнадцать тайм-слотов. Вторая конференция — Demo Day ITMO 2026, мероприятие университетского формата с другой структурой программы: двести десять докладов в семи залах различной вместимости на пятьдесят семь тайм-слотов. Использование двух мероприятий разной структуры позволяет в главе 4 провести проверку переносимости относительного порядка политик между мероприятиями разного формата.

Эмбеддинги описаний докладов и профилей участников получены однократным вызовом внешней модели представления текстов; результаты сохранены в бинарных файлах. Синтетическая популяция участников сформирована однократной LLM-генерацией трёхсот описаний; для воспроизведения эффектов согласованного спроса в реалистичных масштабах популяция расширена до тысячи двухсот участников с помощью добавления гауссовых шумовых составляющих к эмбеддингам базовых участников, что соответствует представлению реальных участников как кластеров вокруг прототипических профилей.

Для проверки переносимости результатов на принципиально иную предметную область использован публичный набор данных «MovieLens 1M», включающий миллион пользовательских оценок около четырёх тысяч кинофильмов, поставленных шестью тысячами пользователей. Кинофильмы преобразованы в сущности, аналогичные докладам конференции, с заданием жанра как идентификатора зала и формированием синтетических тайм-слотов; пользовательские эмбеддинги получены агрегированием жанровых индикаторов посещённых высоко оцененных фильмов. Полученная экспериментальная обстановка не воспроизводит механики реальной конференции, но обеспечивает возможность независимой проверки относительного порядка политик на сущностно ином источнике данных. Подробное описание методики переноса и полученных результатов приведено в главе 4.

## 3.7 Технологический стек и инструментальное обеспечение

Программный комплекс реализован на языке Python версии 3.12 с использованием библиотек NumPy и SciPy для численных вычислений, scikit-learn для обучения вспомогательных моделей, Matplotlib для визуализации результатов, Stable Baselines 3 и расширения для маскируемых действий для реализации обучаемой политики, NetworkX для построения графа знакомств, PyTorch как зависимости библиотеки обучения с подкреплением. Доступ к большим языковым моделям реализован через единый совместимый с OpenAI протокол с маршрутизацией через сервис OpenRouter; в качестве рабочей модели для экспериментов с автономными агентами и оценкой предпочтений использовалась модель Claude Haiku 4.5 разработки Anthropic. Управление виртуальным окружением и зависимостями выполнено с помощью утилиты uv. Хранение версий кода и результатов экспериментов организовано через git с ведением единой основной ветки.

## 3.8 Выводы по главе

В главе описана реализация программного комплекса, поддерживающего сравнительное исследование рекомендательных политик в условиях ограничений вместимости при отсутствии исторических данных взаимодействий. Реализованный комплекс объединяет параметрический симулятор, симуляционную среду на основе автономных агентов с большими языковыми моделями, набор из двенадцати рекомендательных политик различной природы, две модели релевантности и три тестовых набора данных. Дополнительно введены параметры экспериментальной обстановки, отражающие признак популярности докладов и восприимчивость пользователей к рекомендации, а также механизм обмена впечатлениями между агентами во втором слое валидации.

Реализованный комплекс обеспечивает воспроизводимость отдельных этапов сравнительного эксперимента и допускает расширение состава политик и наборов данных без изменения интерфейсных контрактов. Результаты применения комплекса к перечисленным в главе 2 задачам — основное сравнение политик, исследования влияния отдельных компонентов метода и проверка устойчивости результатов, итоговая валидация во втором слое симуляционной среды — представлены в главе 4.
