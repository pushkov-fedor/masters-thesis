**Правило:** перед тем как использовать какой-либо набор данных как «ground-truth» для калибровки/валидации, явно проверить:
1. Что эти данные на самом деле означают (а не только что подсказывает имя файла).
2. Достаточно ли их по объёму для статистики.
3. Совпадает ли источник сигнала с тем, что нужно для эксперимента.

**Why:** в апреле 2026 при обзоре архива Mobius я объявил `pc_votes.xlsx` ground-truth для релевантности и `Registrations.xlsx` ground-truth для посещаемости. Пользователь поправил: pc_votes — это голоса программного комитета о включении доклада в программу (не сигнал релевантности участникам), а в Registrations всего 5 записей — статистически бесполезно. Я сделал поспешные выводы по именам файлов.

**How to apply:**
- Если в данных есть xlsx/csv/json и их планируется использовать как опору — сначала открыть, посмотреть структуру и количество строк, спросить пользователя «правильно ли я понимаю, что X означает Y».
- Не писать в план «использовать как ground-truth» без явного подтверждения.
- Имя файла — слабый сигнал, особенно в командных проектах.
