В Slidev (CommonMark + remark/markdown-it) **markdown и KaTeX не обрабатываются внутри HTML-блока `<div>` без пустых строк** до и после контента. Сырой `**bold**` и `$math$` просачивается в кадр.

**Why:** обнаружено 2026-05-11 при сборке предзащиты ВКР через Slidev + neversink. На черновой версии 4 слайда из 13 оказались с непроработанным markdown/KaTeX. Скриншоты через Playwright MCP сразу показали `$w_{rec}=0$` и `**не уступает**` в виде литералов. Это стандартное поведение CommonMark spec, не баг Slidev.

**How to apply:** всегда писать так:

```html
<div class="p-3 rounded bg-bluegreen-50">

**bold** работает, $w_{rec} = 0$ рендерится через KaTeX

- список тоже работает

</div>
```

Между `<div>` и контентом, и между контентом и `</div>` — обязательно пустая строка. Это распространяется на любые HTML-обёртки (grid, columns, цветные блоки, карточки). Если контент маленький и однострочный (например, иконка или цифра без markdown) — пустые строки не нужны.

Альтернатива — MDC-синтаксис `::div{.class}` ... `::`, но он не нужен, если уже есть `mdc: true` в frontmatter и соблюдены пустые строки.

Дополнительно: `::note::` в `layout: cover` темы neversink интерпретируется как слот темы, а не presenter-note. Для speaker-notes использовать стандартный Slidev `<!-- ... -->` HTML-комментарий после контента слайда.

Перед финальной сборкой PDF — всегда прогон через Playwright MCP по каждому слайду на dev-сервере: `browser_navigate http://localhost:3030/N` → `wait` → `take_screenshot`. Это ловит проблемы за минуты, которые в PDF выглядели бы как «текст плывёт».
