**Правило:** не запускать команды, которые ставят что-либо в систему пользователя на глобальном уровне (brew, apt, системный pip, npm -g и т.д.), даже если это нужно для выполнения задачи. Всегда спрашивать отдельно, либо предлагать альтернативы, не требующие установки.

**Why:** пользователь резко возмутился, когда я попытался `brew install poppler` для парсинга PDF без согласования. Это глобально влияет на его систему, hard-to-reverse, и вне scope того, о чём его просили. Это grave нарушение «action reversibility» из системных инструкций.

**How to apply:**
- Если задача требует недоступного инструмента — **сначала спросить**, ставить ли, или предложить обходной путь (другой инструмент уже в системе, скрипт без зависимостей, `!`-команда для запуска пользователем самому).
- Локальные venv / npm в проекте, `uvx`/`pipx run` и подобные изолированные запуски — тоже сначала уточнить, пусть и менее инвазивны.
- Если вообще без установок нельзя — честно сказать «без X не смогу, установить?» и остановиться.
