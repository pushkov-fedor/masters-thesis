**Правило:** для Python-окружений в этом проекте создавать venv и ставить пакеты через `uv`, не через `pip`/`python -m venv`.

Базовая последовательность:
```
uv venv .venv
source .venv/bin/activate
uv pip install <packages>
```

Или сразу с конкретной версией Python: `uv venv --python 3.12 .venv`.

**Why:** пользователь явно попросил перейти на uv при создании окружения для экспериментов ВКР. uv быстрее pip и надёжнее в плане резолюции зависимостей; пользователь стандартизировал на нём свой стек.

**How to apply:**
- В любом новом Python-проекте/подпроекте — сразу через uv.
- В существующих venv, где уже стоит pip — не ломать, продолжать pip; uv применяется к новым окружениям.
- Если uv не установлен — спросить пользователя перед глобальной установкой (правило о глобальных установках перевешивает).
