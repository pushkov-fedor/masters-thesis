## Банк вопросов комиссии — по трекам

Используй этот банк в шаблоне A: подставь 3 вопроса для трека работы и дай **конкретные ответы применительно к работе студента**, а не общие отписки.

### Industrial

1. **«Вы получили результаты — это хорошо или плохо? С чем сравниваете и почему именно с этим baseline?»**
   → Ответ: указать конкретный baseline (SOTA / коммерческое решение / наивный подход), привести цифры, объяснить выбор именно этого сравнения.
2. **«Что произойдёт, если нагрузка вырастет в 10 раз? Где система сломается?»**
   → Ответ: назвать узкое место (БД, очередь, GPU), стратегию масштабирования, или честно сказать «не тестировали — это ограничение работы».
3. **«Какова стоимость владения системой в год? Когда окупится?»**
   → Ответ: TCO с разбивкой на инфраструктуру / лицензии / поддержку, ROI при заданном масштабе, точка окупаемости.

### Science

1. **«Где ваша система ломается? Покажите 5 худших примеров и объясните почему.»**
   → Ответ: конкретные примеры из тестовой выборки + объяснение природы ошибок (out-of-distribution, ambiguous labels, малая представленность класса).
2. **«Что если убрать компонент X — как это повлияет на метрику?»**
   → Ответ: ablation study с цифрами по каждому компоненту. Если ablation не делали — это критическая дыра.
3. **«Почему именно эта метрика, а не другая? Какие у неё ограничения?»**
   → Ответ: соответствие метрики типу задачи и реальной цели (не «по умолчанию для бенчмарка X»), плюс честный анализ её слабых мест.

### Startup

1. **«Покажите unit-экономику. CAC и LTV сходятся? На каком масштабе?»**
   → Ответ: формула CAC и LTV, конкретные цифры из пилота / гипотез, точка ARR, при которой бизнес становится устойчивым.
2. **«Что не может скопировать конкурент за 3 месяца? Где unfair advantage?»**
   → Ответ: данные / эксклюзивные интеграции / network effect / экспертиза команды — что-то, чего нельзя получить покупкой инженеров.
3. **«Сколько пользователей реально используют продукт регулярно (DAU/WAU/MAU, retention)?»**
   → Ответ: конкретные метрики из пилота, retention curve, причины оттока.

### Education

1. **«Как измерили образовательный эффект? Какой размер выборки и контрольная группа?»**
   → Ответ: pre/post тест, контрольная группа, статистический тест (Wilcoxon / Mann-Whitney для малых выборок), p-value, effect size.
2. **«Если убрать AI-компонент — работа всё ещё квалификационная? Что именно делает AI?»**
   → Ответ: уникальная задача, которую может решить только AI (персонализация, генерация, оценка свободного ответа), и которую не решает обычный LMS.
3. **«Как масштабировать на другие курсы / вузы / возрастные группы?»**
   → Ответ: трансферабельные части (модели, prompts, методология) и привязанные к контексту, план переноса.

## Шаблон D — Инженерный проект без гипотезы (Режим D)

```markdown
**Ситуация:** студент говорит «у меня инженерный проект, гипотезы нет»

**Шаг 1 — уточни:**
«Что именно ты строишь? Опиши одним предложением без технических терминов.»

**Шаг 2 — выяви engineering hypothesis:**
«Твоя система работает лучше существующего решения — в чём именно лучше? Быстрее, точнее, дешевле, масштабируемее?»

**Шаг 3 — сформулируй критерии успеха:**
«При каких измеримых результатах работа считается успешной?»

**Финальный формат:**

Решение [X] обеспечит [критерий успеха: метрика + порог]
в условиях [ограничения / масштаб / бизнес-контекст],
в отличие от [текущего подхода], который [конкретно не справляется].
```

```markdown
- «Комиссия точно спросит [конкретный трек-вопрос]. У меня в работе ответа нет — стоит ли добавить раздел или подготовить устный ответ с запасным слайдом?»
```

> **Примечание для скилла при использовании Шаблона G:**
> - Заполняй ВСЕ разделы — не пропускай критерии, даже если они «н/п». В таком случае пиши «н/п, потому что...»
> - Цитаты — точные, без перефразирования. Если в работе кириллица повреждена при извлечении — извлеки заново с правильной кодировкой.
> - Указывай номера страниц / разделов по возможности — это позволит студенту быстро найти место.
> - Если работа > 80 страниц — отчёт получится длинным (10–20 КБ текста). Это нормально, не сокращай ради краткости.
