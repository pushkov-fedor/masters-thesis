# vkr-plan-checker

Скилл для Claude Code — методологический ассистент дипломника. Помогает проверять и улучшать план ВКР, постановку задачи и гипотезу с учётом новых правил защиты 2026 года.

## Как он работает

7 режимов под разные задачи студента:

- **A** — проверить готовый текст плана / постановки / гипотезы
- **B** — собрать постановку с нуля по вопросам
- **C** — быстрая проверка одной гипотезы
- **D** — инженерный проект без классической метрической гипотезы
- **E** — обзор литературы (7 этапов от исследовательского вопроса до gap)
- **F** — проверка презентации защиты
- **G** — **полная проверка готового текста ВКР** — студент скидывает PDF/docx целиком + указывает тему и трек, на выходе получает сводный отчёт о готовности к защите с цитатами, картой проблем и планом доработки

Прогоняет работу по 12 критериям + бонус за внешнюю валидацию. Выдаёт разбор с цитатами, улучшенную версию формулировки, 3 трек-специфичных вопроса от комиссии и фразы для разговора с научруком.

В Режиме G дополнительно: карта работы по разделам, найденные red flags с цитатами, top-5 сильных сторон, угрозы на защите (Q&A risks), приоритизированный план доработки с оценкой времени.

## Что внутри

- **Чеклист по 12 критериям** с под-чеклистами для 15+ типов задач (RAG, GraphRAG, мультимодальные, CV+edge, anomaly detection, information retrieval с reranking, рекомендательные системы, биометрия, OCR / Document AI, voice-агенты, action recognition, multi-object tracking и др.)
- **79 типичных ошибок** с исправлениями — методологические red flags по разным секциям работы
- **Гайд по двум типам гипотез** (метрическая и архитектурная) с примерами по 4 трекам
- **Анатомия плана** с принципиальными структурными блоками — глоссарий, положения на защиту, научная новизна, анализ рынка, MoSCoW-декомпозиция требований, future work
- **Шаблоны разбора** для всех 6 режимов
- **Банк трек-специфичных вопросов комиссии** для Industrial / Science / Startup / Education

## Как создавался

Базой стал материал семинаров «ВКР без боли» и анализ **21 лучшей ВКР** программы «Искусственный интеллект» ИТМО за 2025 год — полнотекстовый, без сокращений.

Каждая работа разбиралась отдельно: что в ней структурно сильное, какие методические приёмы стоят отдельного внимания, какие ошибки видно по контрасту. Из этого выжато ~150 методологических находок и 79 red flags.

В самом скилле остались только обобщённые методологические принципы — без привязки к конкретным работам, фамилиям выпускников и темам ВКР.

## Установка

Скилл распространяется как архив `vkr-plan-checker.skill`. Загружается в Claude Code одной командой через интерфейс установки скиллов.

## Структура файлов

```
vkr-plan-checker/
├── SKILL.md                    — основной файл с описанием режимов и правилами
└── references/
    ├── checklist.md            — 12 критериев + под-чеклисты по типам задач
    ├── plan-anatomy.md         — анатомия плана + структурные блоки + методологические подходы
    ├── hypothesis-guide.md     — гайд по двум типам гипотез
    ├── examples.md             — примеры слабых и сильных постановок по 4 трекам
    ├── red-flags.md            — топ типичных ошибок с исправлениями
    └── output-template.md      — шаблоны финального разбора (A, B, C, D, E, F)
```

## Контекст применения

Скилл рассчитан на программу «Искусственный интеллект» ИТМО, но методологическая база применима к любым ML / Data / AI-Engineering ВКР. Поддерживает работу с русско- и англоязычными текстами.

## Что изменилось на защите в 2026 году

Скилл закладывает два главных сдвига:

1. **Валидация важнее реализации.** В условиях AI-coding ассистентов «написать работающий пайплайн» перестало быть достаточным доказательством квалификации. Комиссия задаёт вопрос: *«Что именно ты доказал?»*

2. **Слепое рецензирование.** Рецензент получает работу без имени автора и без возможности задать вопросы. Главный фильтр любой формулировки: *«поймёт ли умный специалист из смежной области, зачем эта работа нужна, прочитав только текст?»*
